# Day 3 Notes

## 下午会议笔记

### 1. 区块链的共识机制
共识机制是区块链网络中所有节点达成一致的规则。常见的共识算法包括工作量证明（PoW）和权益证明（PoS）。PoW 通过计算复杂的数学问题来验证交易，安全性高但能耗大；PoS 则根据持币量和持币时间选择验证者，更节能高效。以太坊已于 2022 年完成从 PoW 到 PoS 的过渡（The Merge）。

### 2. 智能合约
智能合约是部署在区块链上的自动执行程序，当预设条件满足时自动触发执行，无需第三方中介。最常用的智能合约平台是以太坊，使用 Solidity 语言编写。智能合约一旦部署便不可篡改，因此安全审计至关重要。

### 3. 钱包与私钥
区块链钱包用于管理加密资产和与 DApp 交互。核心是私钥——一组随机生成的数字，谁持有私钥谁就控制对应地址的资产。助记词（Mnemonic）是私钥的人类可读形式，通常由 12 或 24 个英文单词组成。冷钱包（离线存储）比热钱包（联网）更安全，适合长期持有大额资产。

### 4. 区块链发信息的流程
在区块链上"发信息"本质上是发起一笔交易（Transaction），具体流程如下：
1. **构造交易**：用户指定接收地址、发送金额（或调用合约的数据），并用私钥对交易进行签名。
2. **广播交易**：签名后的交易通过节点广播到整个 P2P 网络。
3. **交易进入 Mempool**：验证节点收到交易后，先验证签名和余额，合法的交易进入内存池（Mempool）等待打包。
4. **打包出块**：矿工/验证者从 Mempool 中选取交易打包进区块，并通过共识机制确认区块。
5. **区块确认**：区块被添加到链上后，交易获得确认。随着后续区块的增加，交易被篡改的可能性指数级下降。

### 5. 什么是 RPC
RPC（Remote Procedure Call，远程过程调用）是一种通信协议，允许客户端程序调用远程服务器上的函数，就像调用本地函数一样。在区块链中，RPC 是 DApp 与区块链节点交互的核心桥梁：

- **JSON-RPC**：区块链最常用的 RPC 实现，使用 JSON 格式传输数据，通过 HTTP 或 WebSocket 通信。
- **典型用途**：查询账户余额、发送交易、读取合约状态、监听链上事件等。
- **RPC 端点**：通常由节点服务商（如 Infura、Alchemy）或自建节点提供，形如 `https://mainnet.infura.io/v3/YOUR-KEY`。
- **工作流程**：前端 → JSON-RPC 请求 → 区块链节点 → 返回链上数据 → 前端展示。

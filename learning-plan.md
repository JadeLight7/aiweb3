# Personal Learning Plan

> 基于 [Handbook 四层知识地图](https://aiweb3.school/zh/handbook/) 构建。优先问题驱动路径，而非线性阅读。

## Profile Summary

- Web3 熟练 + 编程独立开发 + AI 有基础
- 目标：Hackathon 项目产出
- 每日时间：1-2 小时

## Focus Path: On-Chain Agent Builder

```
Agent → Web3 Tool Use → Agent Workflow → Agent Wallet → Machine Payment → Verifiable AI
```

## Phase 1: Foundation Alignment (Week 1-2)

补齐 AI × Web3 交叉基础，建立 shared language。

| Week | Topic | Key Tasks |
|------|-------|-----------|
| W1 | Agent + Web3 Tool Use | 搭建 Learning Agent；理解 Agent 如何调用 RPC / Wallet / Contract |
| W2 | Agent Wallet + Permissions | Session Key、Smart Account、权限边界实验 |

### Phase 1 Checkpoints
- [ ] Learning Agent 正常运行
- [ ] Agent 成功发起至少 1 笔链上交互（testnet）
- [ ] Agent Wallet 权限模型 demo

## Phase 2: Bridge Deep-Dive (Week 3-4)

深入交叉领域，产出可复用模块。

| Week | Topic | Key Tasks |
|------|-------|-----------|
| W3 | Machine Payment + Settlement | Agent 间支付、托管、结算原型 |
| W4 | Verifiable AI + AI Security | 模型输出上链验证、Prompt Injection 防护 |

### Phase 2 Checkpoints
- [ ] Agent-to-Agent 支付流程跑通（testnet）
- [ ] 至少 1 个可验证 AI 输出场景实现
- [ ] Security 边界分析文档

## Phase 3: Hackathon Prep (Week 5-6)

聚焦 Hackathon 赛道，快速原型迭代。

| Week | Topic | Key Tasks |
|------|-------|-----------|
| W5 | Track 选型 + 原型 | 确定 Hackathon 赛道，搭建最小可用原型 |
| W6 | 打磨 + 提交 | 完善 demo、文档、pitch deck |

### Phase 3 Checkpoints
- [ ] Hackathon 赛道确定
- [ ] MVP 原型可演示
- [ ] 提交材料准备完成（demo video / repo / deck）

## Daily Routine

1. 阅读 WCB Learning 页当天课程 / 任务
2. 对照 Handbook 对应节点，生成 minimum / recommended / challenge 路径
3. 产出 `daily/YYYY-MM-DD.md`
4. 生成打卡草稿，手动提交到 WCB
5. 遇到 Handbook 问题 → 记录到 `handbook-feedback/`
